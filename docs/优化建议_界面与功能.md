# 优化建议（界面 & 功能）

日期：2026-05-29

## ⚠️ 已发现的 Bug：导航栏高亮失效
`base.html` 用 `request.endpoint` 判断当前页，但 Blueprint 重构后端点变成
`main.dashboard`/`products.menu` 等带前缀形式，模板里仍在比对旧的扁平名
（`dashboard`/`menu`…），导致导航下划线高亮全部失效。需修复判断逻辑。

## 界面建议
1. 修复导航高亮；并做窄屏/移动端适配（汉堡菜单或换行）。
2. 各列表空状态友好提示 + 引导按钮，而非空白表格。
3. 删除操作加二次确认（confirm 弹窗），防误删。
4. 提交类操作（下单/改状态）按钮禁用 + “处理中…”，防重复提交。
5. 统计页挑 1~2 个关键报表做可视化（纯 CSS 柱状条即可），答辩观感强。
6. 仪表盘数字卡片加环比小箭头（今日 vs 昨日）。

## 功能建议
1. 给菜单、库存也加搜索/分类筛选，体验与会员页一致。
2. 订单退款/作废流程。
3. 低库存预警提示放到仪表盘顶部（用到多表关系）。
4. 会员积分可兑换/抵现，形成闭环。
5. 统计页导出 CSV。
6. 操作日志表（谁、何时、改了什么）——契合“多表关系”主题。

## 推荐优先级
先修导航高亮 bug + 删除二次确认（改动小、立竿见影），
再考虑统计页图表可视化（答辩加分）。
