# 代码质量优化：完成第 7、8 项

完成日期：2026-05-29

## 第 7 项：修复 N+1 查询

订单列表、仪表盘等页面原先在循环里逐条访问关联对象（顾客、员工、商品），
导致一次页面渲染触发几十上百条 SQL。已用预加载消除：

- **一对一关联**（订单 → 顾客 / 员工）：用 `joinedload`，一条 JOIN 取回。
- **一对多关联**（订单 → 订单明细）：用 `selectinload`，一条 IN 查询批量取回。

效果：订单/仪表盘页面的 SQL 数量从「随记录数线性增长」降为「固定几条」。

## 第 8 项：把 1000 行 app.py 拆成 Blueprint

原来所有路由都堆在一个 `app.py` 里（约 1000 行），难维护。已重构为：

| 文件 | 职责 |
|------|------|
| `src/app.py` | 仅应用工厂：建应用、加载配置、初始化数据库、注册蓝图、模板上下文 |
| `src/decorators.py` | `login_required` / `admin_required` 装饰器、`prod_emoji` 辅助函数 |
| `src/bootstrap.py` | `bootstrap_demo_data()`：演示数据初始化 |
| `src/routes/__init__.py` | `register_blueprints(app)`：统一注册全部蓝图 |
| `src/routes/auth.py` | 登录、登出、切换账号 |
| `src/routes/main.py` | 仪表盘 |
| `src/routes/products.py` | 菜单、商品增删改 |
| `src/routes/orders.py` | 订单列表、新建、详情、改状态 |
| `src/routes/customers.py` | 顾客增删改查 |
| `src/routes/staff.py` | 员工管理 |
| `src/routes/inventory.py` | 库存 |
| `src/routes/promotions.py` | 促销活动 |
| `src/routes/recipes.py` | 配方（商品↔原料） |
| `src/routes/analytics.py` | 统计分析（含 8 个多表 JOIN 报表） |
| `src/routes/api.py` | JSON 接口 |

配套改动：所有模板的 `url_for()` 加上蓝图前缀（如 `url_for('orders.order_detail')`），
共 15 个模板、53 处替换，由一次性脚本 `update_template_urls.py` 完成。
