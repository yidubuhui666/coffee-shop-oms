# 优化建议（第二轮）

日期：2026-05-29
前提：N+1 查询修复（#7）与 app.py 拆分 Blueprint（#8）已完成。

## P0 — 投入小、收益明显

### 1. 列表分页
`/orders`（150+ 条）、`/customers`、`/menu` 目前一次性全查全渲染，数据多会变慢。
用 Flask-SQLAlchemy 的 `.paginate()` + 模板分页条解决，改动小、演示直观。

### 2. 全局错误页（404 / 500）
现在出错显示 Flask 默认报错页。加 `@app.errorhandler(404/500)` + 简单 `404.html`/`500.html`。

### 3. 整理一次性脚本
根目录/`src/` 下有跑完即弃的脚本：`update_template_urls.py`、`map_to_unique_images.py`、
`revert_to_safe_images.py`、`gen_images.py`、`scrape_*.py`、`batch_ai_*.ps1`。
建议挪进 `tools/` 或删除。**注意保留所有 `seed_*.py`**（数据同步要用）。

## P1 — 中等投入、提升工程质量

### 4. 基础测试
新增 `tests/test_smoke.py`：用 SQLite 内存库验证「各页登录后 200」「下单后库存正确扣减」。
课程项目加分项，也防止以后改坏。

### 5. 配置清理
`config.py` 删掉 `root:root` 默认连接串兜底，改为缺 `DATABASE_URL` 时报错提示；
`SECRET_KEY` 同样强制走环境变量。

## P2 — 可选

### 6. 密码加密存储
`staff.password` 现为明文。用 Werkzeug（已在依赖里）的
`generate_password_hash`/`check_password_hash`。需配套幂等迁移脚本把现有明文转哈希。

### 7. 模型层抽公共方法
订单总价计算、会员折扣等散落在路由里，挪到 `models.py` 做成方法（如 `order.recalc_total()`），
让路由更薄、逻辑集中。

## 推荐先做

1 + 2 + 3 一次可完成，对「展示多表查询关系」的课程项目最实用。
